# vibe-coding-docs 文档结构规范

## 目录结构

```
docs/
├── plan/
│   └── plan.md                 # 项目主计划文档
├── archive/
│   ├── v1.0/
│   │   ├── 2025-01-15-变更记录.md
│   │   └── 2025-01-20-测试记录.md
│   └── v1.1/
│       └── 2025-02-01-发布记录.md
└── technical/
    ├── v1.0/
    │   ├── 2025-01-10-认证方案.md
    │   └── 2025-01-18-Bug修复.md
    └── v1.1/
        └── 2025-02-05-数据库设计.md
```

## 文档命名规范

### 版本目录
- 格式: `v{MAJOR}.{MINOR}`
- 示例: `v1.0`, `v1.1`, `v2.0`

### 文档文件
- 格式: `YYYY-MM-DD-{标题}.md`
- 示例: `2025-01-15-变更记录.md`

## 文档内容规范

### plan.md 结构
1. 版本信息
2. 已实现内容
3. 待实现内容
4. 项目规划
5. 相关文档链接

### archive 文档结构
1. 标题（包含日期和版本）
2. 概述
3. 详细内容
4. 测试结果（如适用）
5. 备注

### technical 文档结构
1. 标题（包含日期和版本）
2. 问题描述
3. 解决方案（包含代码示例）
4. 相关文件
5. 测试验证
6. 参考资料

## 内容拆分原则

当单个文档超过 800 行时，考虑拆分：

1. **按主题拆分**: 将不同主题的内容分离到独立文档
2. **按时间拆分**: 将长时间跨度的内容按阶段分离
3. **使用相对路径引用**: 在主文档中引用子文档

示例：
```markdown
## 详细实现

详见: [数据库设计详情](./database-design.md)
详见: [API接口文档](./api-spec.md)
```

## CLAUDE.md 文档索引

在项目根目录的 CLAUDE.md 中维护以下章节：

```markdown
## 项目文档

### 文档结构
- [项目计划](docs/plan/plan.md) - 当前版本、已实现、待实现内容
- [历史记录](docs/archive/) - 按版本归档的变更和测试记录
- [技术文档](docs/technical/) - 按版本归档的技术细节和方案

### 快速导航
#### v{VERSION}
- 计划: [plan.md](docs/plan/plan.md)
- 变更记录: [变更列表](docs/archive/v{VERSION}/)
- 技术文档: [技术列表](docs/technical/v{VERSION}/)

#### 历史版本
- v{PREV_VERSION}: [归档](docs/archive/v{PREV_VERSION}/) | [技术](docs/technical/v{PREV_VERSION}/)
```
